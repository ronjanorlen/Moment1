Arbetsprocess

Skapa nytt npm-projekt

Jag skapar en ny katalog lokalt på min dator som döps till "Moment 1", som jag sedan öppnar i kommandoterminalen. I kommandoterminalen anger jag npm init för att initiera ett nytt projekt. Därefter använder jag kommando code . för att öppna detta i Visual Studio Code.

I Visual Studio Code har det då skapats en fil som heter package.json som jag konfigurerar. Jag anger source och src/index.html, browserslist för bakåtkompabilitet samt två olika script för att starta projektet från terminalen.

Installera paket

Jag installerar sedan paketet Parcel med kommando npm install parcel --save-dev för att spara i utvecklings-läget. När paketet är installerat skapas en mapp som heter node_modules/ som innehåller massor av olika paket för att hantera olika åtgärder som sker på webbplatsen i bakgrunden. För att inte versionshantera denna fil lägger jag in node_modules/ i .gitignore-filen.

Bygga webbplats

I Visual Studio Code skapar jag en src-mapp som innehåller css-fil, html-filer, bilder samt JavaScript-fil.

Under utveckling av webbplatsen använder jag kommando npm run start för att starta en live server där jag kan se vilka ändringar jag gör. När ändringar sparas skapas en mapp som heter dist som blir synlig i Visual Studio Code, där alla färdiga filer hamnar och optimeras vid testkörning. "dist"-katalogen samt .parcel-cache läggs också in i .gitignore-filen för att exkluderas vid versionshantering.

Jag gör även commits under tiden jag utvecklar, då använder jag mig utav kommandona git add, git commit -m och git push.

Utvecklingen sker i en separat branch som heter "development" för att skilja på utveckling och publicering. Publicering sker sedan från main-branchen.
